在 Github 使用 Hexo 架設部落格

前置作業

  • Nodejs
    需要使用到 npm
    在終端機輸入以下指令，檢查是否安裝

  • Git
    需要使用 Git 推送程式碼及資料到 Github

開始正題

  1. 在 GitHub 建立一個 repository

  2. 安裝 Hexo 在全域

  3. 初始化一個 Hexo 資料夾（你也可以叫他專案)

  4. 安裝需要的套件

  5. 配置 yml 設定檔

Site 網站配置

配置屬性 說明
title 網站名稱

Url 網站路徑配置

配置屬性 說明
url 網站網址
ex: http://github_user_name.github.io/
root 網站的根目錄，github 的 repository 名稱
ex: blog

Deployment 部署配置

配置屬性 說明
deploy 部署的意思
type 部署的方式
ex: git
repo git 上面的 repository 路徑，從 github repository 的 use ssh 取得
ex: git@github.com:github_user_name/blog.git
branch github 的分支名稱
ex: master

  1. 初始化為 git 專案

  2. 在專案下設定 Github 專案的遠端路徑

$ hexo generate

$ hexo deploy
```
